This book teaches modern full-stack architecture by reading one real system closely: rmhstudios.com. Rather than survey patterns in the abstract, it follows how a single codebase grew from a starter template into a process-split platform with server-side rendering, a fleet of realtime services, a large relational data layer, an in-browser games library, and a live migration toward Kubernetes.

Who this book is for

The intended reader is a computer-science student who has written some web code — enough to recognize HTTP, a database query, and a React component — but who has not yet seen how the pieces fit together at the scale of a production platform. No prior knowledge of TanStack Start, Prisma, Socket.io, Go, or Kubernetes is assumed; each is introduced from first principles when the case study first needs it.

How each chapter works

Every chapter follows the same rhythm so you always know where you are:

  • Learning objectives open the chapter — the concrete things you should be able to do or explain afterward.
  • The body develops the ideas with diagrams (every figure is numbered), real code excerpts from the repository, and key-term callouts that define vocabulary the first time it appears.
  • A summary distills the chapter to a handful of load-bearing ideas.
  • Practice exercises close the chapter. Each is numbered chapter.item (for example, 5.3) and tagged by type and difficulty.

Reading the exercise tags

Exercises carry two badges. The first names the kind of thinking required; the second estimates effort.

Type tagWhat it asks you to do
conceptExplain or compare an idea in your own words.
traceFollow a request, event, or value through the system step by step.
diagramRead, complete, or draw an architecture diagram.
designMake and defend a design decision under stated constraints.

Difficulty is one of easy, medium, or hard.

Practice exams and solutions

The back matter contains three practice exams — a Midterm covering Chapters 1–7, a Final covering Chapters 8–13, and a Comprehensive exam spanning the whole book. Each states a time budget and point allocation. Appendix A gives worked solutions to every chapter exercise; Appendix B gives full answer keys for the exams. Use them honestly: attempt first, check second.

A standing promise about accuracy

Grounded in the repository. Ports, file paths, model names, event names, and scripts in this book are drawn from the actual codebase. Where a subsystem is a scaffold or an in-progress migration rather than a finished, battle-tested component, the text marks it with a scope note instead of overstating it.

Typographic conventions

  • monospace marks code: identifiers, file paths, commands, ports.
  • Blue note, amber warning, green tip, and purple scope callouts highlight, respectively, useful asides, hazards, good practice, and honest limits on what a component actually does.
  • Figures are numbered by chapter (Figure 7.2 is the second figure in Chapter 7).
