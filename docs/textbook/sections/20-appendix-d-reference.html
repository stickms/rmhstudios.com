This appendix is a flat reference: the processes and the ports they bind, the realtime event vocabulary the fleet speaks, a map of the repository's top-level directories, and the handful of environment variables that hold the system together. Everything here is read directly from the codebase — docker-compose.yml, the package.json scripts, the server/ handlers, and .env.example. Where a name in the prose chapters was approximate (Baccarat's prefix, for example), this table gives the exact string as it appears in source. Use it as a lookup, not a tutorial; the narrative chapters explain why each piece exists.

Processes & Ports

The platform builds a single Docker image and runs it as many containers, each launching one entrypoint of the form node dist-server/server/<name>/index.cjs. In development, pnpm run dev launches the same processes under tsx watch. The ports below are the production defaults baked into docker-compose.yml (${PORT_*:-default}); the compose header documents the staging set as 8005 / 8001 / 8676 / 8003. Every published port binds to 127.0.0.1 only and is reached publicly through Apache path-routing.

Non-application ports. Compose also runs minio (S3-compatible object storage) on 9000 with its console on 9001, plus a one-shot minio-init that creates the bucket and exits. These are infrastructure, not part of the realtime fleet.

Realtime Event Catalog

Each realtime service speaks an event-driven protocol: named events with JSON payloads, socket.on on one side and emit on the other. Games are kept disjoint by event-name prefix rather than by Socket.io namespace. Below, C2S means client→server (a socket.on handler) and S2C means server→client/room (an emit or io.to(room).emit). The lists are representative, not exhaustive; event names are quoted exactly as they appear in source.

A few prefixes differ from what a reader might guess: Baccarat is bacc: (not baccarat:), Roulette is rl: (not roulette:), Lights Out is hyphenated lights-out:, and Slice It uses bare event names (join_lobby, score_update) with no prefix, scoping by raw lobby-id rooms. The recap service emits no Socket.io events at all — it is a Discord-gateway client.
