Before this document was written, the entire Go fleet went through a three-part senior engineering review — one reviewer per subsystem — exactly as a real pre-merge review would. The reviewers found genuine issues, including a blocking concurrency bug and a real data race. This section records the findings and the fixes honestly, because the remediation loop is part of the work.

The review process

Three independent reviews ran in parallel: (1) the shared foundation + gateway, (2) the four realtime services, (3) the workers + discord-bot + the deploy layer. Each reviewer read the code directly and reported prioritized findings (blocking / high / medium / low) with concrete fixes. Findings were then fixed and the full verification suite re-run, including the race detector.

Blocking findings (fixed)

FindingFix
Send-on-closed-channel panic. Conn.close() closed the send channel while other goroutines (heartbeats, timers, broadcasts) could still write to it — a guaranteed crash under exactly the load realtime is built for.Reworked the connection lifecycle: a dedicated closed chan struct{} (closed once) signals shutdown; send is never closed; all producers go through trySend, which selects on closed. The write pump exits on closed.
Gateway upstreams unresolvable in k8s. The Helm chart didn't set the gateway's *_UPSTREAM env, so its compose-style defaults (e.g. http://gamehub:7001) wouldn't resolve to the in-cluster Service names.The chart now injects the correct in-cluster DNS (http://rmhstudios-go-gamehub:7001, etc.) plus WEB_UPSTREAM for the gateway Deployment.
CI/script contract mismatch. CI invoked run.sh --no-docker, which the script didn't support.Added a real --no-docker mode that builds + launches the binaries on the runner against the CI-provided Postgres.

The data race the detector caught

The most instructive finding came not from a human read but from go test -race: in rmhbox's StateSync, the timer-broadcast goroutine read h.stop in its select while TimerHandle.Cancel() set h.stop = nil under the mutex — an unsynchronized read/write on the same field.

// before: Cancel niled the channel the goroutine was reading
t.stop = nil                       // RACE vs the select below
// goroutine:
select { case <-h.stop: return ... }

// after: never nil it; capture once; close-once is the signal
stop := h.stop                     // captured before goroutine starts
go func(){ for { select { case <-stop: return ... } } }()
// Cancel(): close(t.stop) guarded by a cancelled flag; no nil
Outcome: after the fix, go test -race ./... is clean across every package. This is the value of running the detector in CI: a bug that is invisible to the eye and intermittent in production was caught deterministically.

High-severity findings (fixed)

FindingFix
discord-bot /rmhbot session lock was a check-then-set TOCTOU race; two interactions could run two agent loops on one worktree.Per-session mutex; atomic tryLock/unlock; history mutated only under the lock.
vibe-worker built the data: URL by string concatenation; arbitrary HTML with #/% truncated or mis-decoded in Chromium.Base64-encode the payload (data:text/html;base64,…).

Selected medium findings (fixed)

  • rmhbox grace timers captured a stale lobby pointer and didn't cancel preload/ingame keys on force-end → re-resolve the live lobby in each closure; cancel all grace keys in clearLifecycleTimers.
  • rmhbox pause/resume used a concrete type-switch → replaced with a baseGame() interface method the compiler enforces.
  • rmhtube didn't prune skip/queue votes on disconnect and disbanded outside the room lock → prune on member removal; disband under the lock with a join re-check.
  • rmhmusic onPlay released and re-acquired the lock, allowing interleaved track changes → single critical section; seq stamping aligned with rmhtube.
  • workers logged-and-continued on a metrics-port bind failure, leaving pods unprobeable → fail fast on bind error.
  • auth treated DB errors as "anonymous" silently → log/meter non-authentication errors.
  • discord-bot ran interactions on context.Background() → derive from the lifecycle context with a timeout; make the GitHub repo env-driven.

What the reviewers checked and found clean

Not everything needed fixing. The reviewers explicitly signed off on: the auth SQL (parameterized, correct expiry), the pgx pool configuration, graceful shutdown, the doctrine determinism (golden-tested), the rmhbox atomic SELECT … FOR UPDATE stat merge, per-player state scoping (no info leak), the safePath traversal guard, and the consistent room→manager lock ordering across the realtime services. The Dockerfile (static build, tzdata, non-root) and the Helm probe-port helper were also verified correct.
