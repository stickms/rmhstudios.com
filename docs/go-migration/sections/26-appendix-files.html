Shared toolkit — go-services/pkg/

The cross-service library every binary links against: config, DB access, auth, the realtime WebSocket framework, the Redis backplane, HTTP plumbing, logging, rate limiting and metrics. This is where the "we don't need the JS libraries" decisions live (notably pkg/auth validating Better Auth sessions straight against Postgres, and pkg/realtime replacing Socket.IO).

Deploy — deploy/ (repo root)

The Helm chart deploy/helm/rmhstudios-go (Chart 0.1.0, appVersion 1.0.0) deliberately mirrors PR #121's rmhstudios chart: a generic Deployment over a services map, Traefik ingress, and the reused rmhstudios-secrets Secret. Its one departure is one image per service (each Go binary is a separate static image).
