Verification was treated as a first-class deliverable, not an afterthought. The fleet is validated at four levels — compilation, static analysis, race-tested unit/integration tests, and end-to-end tests against real infrastructure — and every level is green.

The test pyramid

Build, vet, format

The whole module compiles (go build ./...), passes go vet ./... with no findings, and is gofmt-clean (CI fails on any unformatted file).

End-to-end tests against real infrastructure

The e2e suite (build tag e2e) is the proof that the services actually work together, not just that the units pass. scripts/e2e/run.sh starts a real PostgreSQL, loads the schema, builds and launches the real gamehub and rmhtube binaries, waits for their /health, then runs Go tests that connect as real websocket clients.

Infrastructure validation

Beyond Go: helm lint passes, the rendered manifests pass kubeconform -strict (35/35 valid), and the container image builds and runs. The verification toolchain itself runs through Docker (the pinned golang:1.23 image) so it is reproducible on any machine with Docker, with no local Go install required.
