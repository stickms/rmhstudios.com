The realtime event catalog for the four WebSocket services. Every name below is taken from the actual source — the events.go constant maps (rmhmusic, rmhtube), the kk* constants in internal/gamehub/kk.go, and the literal event strings in internal/rmhbox/*.go. The names carry their service prefix (kk:, rmhmusic:, rmhtube:, rmhbox:) and are kept byte-for-byte identical to the legacy TypeScript C2S/S2C maps so the existing client adapters map cleanly onto the new transport.

The common envelope

Every frame in both directions is one JSON object — the realtime.Envelope defined in pkg/realtime/protocol.go. This replaces the Socket.IO v4 wire protocol while preserving its semantics: named events plus the sequence-numbered mutation broadcasts the Node delta protocol relied on.

{ "event": "rmhtube:room:join", "payload": { ... }, "seq": 42, "ts": 1718700000000 }
FieldTypeMeaning
eventstringThe prefixed event name (the rows below).
payloadJSON (omitempty)Event-specific body; absent for e.g. heartbeats.
sequint64 (omitempty)Server-assigned, monotonic per room for mutation broadcasts (ordering guarantee).
tsint64 (omitempty)Server send time in epoch milliseconds.
The *:room:action / rmhbox:game:action broadcasts are the seq-numbered "delta" frames: their payload carries a type discriminator (e.g. MEMBER_JOINED, CHAT_MESSAGE, NOW_PLAYING) plus the mutation data, reproducing the legacy delta protocol.

gamehub — Kowloon Knockout (kk:)

A 1v1 host-authoritative relay: host creates a room, a guest joins, then input (guest→host) and game_state (host→guest) frames are relayed; a fighter_ready handshake from both seats re-arms a rematch. Defined in internal/gamehub/kk.go.

Client → Server

EventMeaning
kk:create_roomHost creates a room with its chosen fighter class.
kk:join_roomGuest joins by room code with a fighter class.
kk:inputGuest input frame relayed to the host (authoritative sim).
kk:game_stateHost's authoritative game-state frame relayed to the guest.
kk:fighter_readySeat signals ready for the (re)match handshake.
kk:leaveLeave the room.

Server → Client

EventMeaning
kk:room_createdAcknowledges room creation; carries the code.
kk:room_joinedAcknowledges a successful join.
kk:opponent_readyThe other seat is ready (rematch handshake progress).
kk:opponent_disconnectedThe peer dropped.
kk:errorError (bad code, room full, etc.).
Only Kowloon Knockout is ported verbatim. The remaining games (Slice It, Neon Driftway, ...) are left as compiling RelayGroup TODO registrations and emit no events yet.
