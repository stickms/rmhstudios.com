Engineering Report

Migrating rmhstudios.com to Go Microservices

A complete design, implementation, and verification record for the Go rewrite of the realtime and worker tier — built on, and reconciled with, the k3s / Helm / Terraform infrastructure of PR #121.

9 services 1 shared module Go 1.23 pgx · gorilla/ws · discordgo · chromedp Helm + Redis backplane build · vet · race · e2e ✓

Scope. The Node/TypeScript realtime & worker processes (socket-server, rmhbox, rmhtube, rmhmusic, recap, doctrine-worker, vibe-worker, discord-bot) re-implemented as compiling, tested Go services, plus an edge gateway — deployable beside the React SSR app via a Helm chart that reuses PR #121's conventions.

Honesty note. PR #121 explicitly declined a Go rewrite as an architecture decision. This work deliberately overrides that decision on an isolated branch. Where a component is genuinely not portable (React SSR, Better Auth issuance) it is kept in JavaScript and integrated, not faked; where a service is partially ported, this document says so plainly.

Status: everything in this report compiles, passes go vet, passes unit tests under the race detector, and passes an end-to-end suite against a real Postgres and real websocket clients.
