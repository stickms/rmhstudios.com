The gateway service is the single edge / backend-for-frontend (BFF) that fronts the whole rmhstudios platform. It listens on PORT_WEB (default 7005) and is the one ingress target the Helm chart from PR #121 routes to: it reverse-proxies the React SSR / Nitro app and the Go realtime services (WebSocket pass-through included), turns a Better Auth session into a trusted identity header set, optionally serves built static assets, and exposes /health + /metrics. It is implemented in internal/gateway/proxy.go, internal/gateway/auth_middleware.go, and wired in cmd/gateway/main.go.

Role: the single edge / BFF

In the Node deployment each browser request could fan out to several processes: the React SSR app, the unified Socket.IO games server, and the rmhbox / rmhtube / rmhmusic realtime services. The gateway collapses that fan-out into one http.Handler. Everything the platform exposes to the public network arrives here first, which makes the gateway the natural place to put two cross-cutting concerns the individual services should not each re-implement: upstream routing and the authentication trust boundary.

The package doc comment states the contract directly: the gateway is “the single ingress target the Helm chart (PR #121) routes to: it fronts the React SSR/Nitro app and reverse-proxies the WebSocket path prefixes to the Go realtime services.” Because it is the only thing the chart needs to publish, the rest of the Go services can stay ClusterIP-internal and trust the identity headers the gateway asserts.

The default upstream targets mirror the Compose / Helm service names and the PORT_* family the Node services already used, so the gateway drops in without renaming anything:

const (
    defaultWebUpstream      = "http://web:3000"
    defaultGamehubUpstream  = "http://gamehub:7001"
    defaultRmhboxUpstream   = "http://rmhbox:7676"
    defaultRmhtubeUpstream  = "http://rmhtube:7003"
    defaultRmhmusicUpstream = "http://rmhmusic:7002"
)

Longest-prefix reverse-proxy router

The router selects an upstream by longest-matching path prefix, falling back to the web (SSR) upstream for everything no Go service owns. Each upstream is a prefix string paired with a parsed *url.URL target and a *httputil.ReverseProxy. NewRouter builds the web catch-all first (prefix ""), then the four Go realtime routes:

routes := []struct {
    prefix string
    target string
}{
    {"/socket/", cfg.GamehubUpstream},
    {"/rmhbox-ws/", cfg.RmhboxUpstream},
    {"/rmhtube-ws/", cfg.RmhtubeUpstream},
    {"/rmhmusic-ws/", cfg.RmhmusicUpstream},
}

Each upstream is configurable at runtime through its *_UPSTREAM environment variable (WEB_UPSTREAM, GAMEHUB_UPSTREAM, RMHBOX_UPSTREAM, RMHTUBE_UPSTREAM, RMHMUSIC_UPSTREAM), resolved by LoadConfig against the defaults above. The realtime hubs serve these exact prefixes (for example the rmhbox hub registers ServeWS on /rmhbox-ws/), so the gateway proxies the full path through unchanged — there is deliberately no prefix stripping.

After registration the prefixed routes are sorted longest-first so selection is deterministic and “most specific wins”:

sort.Slice(r.prefixed, func(i, j int) bool {
    return len(r.prefixed[i].prefix) > len(r.prefixed[j].prefix)
})

The hot path is pick, which walks the sorted prefixes and returns the first match, or the web catch-all when nothing matches:

func (r *Router) pick(path string) *upstream {
    for _, u := range r.prefixed {
        if hasPrefix(path, u.prefix) {
            return u
        }
    }
    return r.web
}

WebSocket upgrade pass-through

The realtime routes all carry WebSocket traffic, and the gateway passes the upgrade through transparently. The key is not doing anything clever: newProxy builds a plain httputil.NewSingleHostReverseProxy and leaves the default Director in place. Go 1.23's httputil.ReverseProxy detects a WebSocket upgrade request, switches into bidirectional-copy mode on a 101 Switching Protocols response, and manages the hop-by-hop Connection/Upgrade headers itself.

The code comment is explicit that this is intentional: “We deliberately do NOT install a Director that strips the Connection/Upgrade hop-by-hop headers… Leaving the default Director in place is exactly what makes WebSocket pass-through work.” A custom Director that scrubbed those headers would silently break the upgrade.

The only customization is an ErrorHandler that logs the upstream target and path and returns a 502 Bad Gateway (“upstream unavailable”) rather than letting the proxy emit its default error.

Auth trust boundary

Because the gateway is the only public ingress, it is also the trust boundary for identity. AuthMiddleware (in auth_middleware.go) wraps the proxy router so every proxied request carries a validated identity header set — or none, for anonymous traffic. It does two things, in order.

1. Strip every inbound X-Rmh-* header

Before any validation, the middleware deletes all client-supplied headers under the X-Rmh- prefix, canonicalizing each name first:

for name := range r.Header {
    if strings.HasPrefix(http.CanonicalHeaderKey(name), trustedHeaderPrefix) {
        r.Header.Del(name)
    }
}
This prefix-wide strip is a hardening step that came out of senior review. Deleting only the three headers the gateway injects would still let a client smuggle a future X-Rmh-* header that some later upstream learns to trust. Stripping the entire prefix means “neither today's headers nor any future X-Rmh-* header can be forged by a client,” as the constant's doc comment puts it.

2. Validate the session and inject trusted identity

If a session token is present (extracted by httpx.SessionToken) and it validates against the SessionValidator, the middleware injects three trusted headers toward the upstream:

if id, err := v.ValidateSession(r.Context(), token); err == nil {
    r.Header.Set(HeaderUserID, id.UserID)   // X-Rmh-User-Id
    r.Header.Set(HeaderName, id.Name)       // X-Rmh-User-Name
    if id.IsAdmin {
        r.Header.Set(HeaderIsAdmin, "true") // X-Rmh-Is-Admin
    } else {
        r.Header.Set(HeaderIsAdmin, "false")
    }
}

The middleware is non-blocking: an absent or invalid token simply passes through as anonymous (the headers stay stripped), because the SSR app renders its own auth pages and the realtime hubs run soft-auth. The SessionValidator is a narrow interface (ValidateSession(ctx, token) (auth.Identity, error)) — the real implementation is *auth.Validator backed by the DB pool, while tests supply a fake, so the middleware is exercisable with httptest alone.

For routes that must be authenticated, the package offers the opt-in RequireAuth helper, which returns a 401 JSON error when no gateway-asserted X-Rmh-User-Id is present. It is an explicit seam for future Go API routes; the proxy catch-all does not use it today.

Static asset serving

The gateway can optionally serve built React assets directly. When STATIC_DIR is set and points at an existing directory, NewRouter installs an http.FileServer over it; otherwise it logs a warning and ignores the value. In ServeHTTP the static server is consulted only after the Go prefixes and only for real files:

func (r *Router) ServeHTTP(w http.ResponseWriter, req *http.Request) {
    if u := r.pick(req.URL.Path); u != r.web {
        u.proxy.ServeHTTP(w, req)
        return
    }
    if r.static != nil && r.staticAssetExists(req.URL.Path) {
        r.static.ServeHTTP(w, req)
        return
    }
    r.web.proxy.ServeHTTP(w, req)
}

staticAssetExists opens the path under the static dir and returns true only for a regular (non-directory) file. This short-circuit-only-for-real-files rule is deliberate: unknown routes — SSR pages and client-side routes — must still fall through to the web upstream instead of 404ing from the file server.

Service wiring (cmd/gateway/main.go)

The entrypoint loads common config, opens the DB pool that backs the auth validator, builds the router, and composes the auth middleware in front of it:

router, _ := gateway.NewRouter(gateway.LoadConfig(), logger)
proxied := gateway.AuthMiddleware(v)(router)

mux := http.NewServeMux()
mux.HandleFunc("/health", httpx.Health("gateway", nil))
mux.Handle("/metrics", metrics.Handler())
mux.Handle("/", proxied) // catch-all -> auth-wrapped proxy router

addr := ":" + config.GetString("PORT_WEB", "7005")

Only /health and /metrics are gateway-local; everything else is handed to the auth-wrapped proxy router, so every proxied request carries the trusted identity headers (or none). The server is run with a 30-second graceful shutdown via httpx.NewServer.

Tests

The gateway is covered at the two seams that matter:

  • Prefix selection. pick is unit-tested for correct longest-prefix routing — each /socket/, /rmhbox-ws/, /rmhtube-ws/, /rmhmusic-ws/ path resolves to its own upstream, and unmatched paths fall to the web catch-all.
  • Header inject / strip. The auth middleware is exercised with a fake SessionValidator over httptest: a valid token injects X-Rmh-User-Id/Name/Is-Admin, while a forged inbound X-Rmh-* header is stripped and never reaches the next handler — the property the trust boundary exists to guarantee.
The fake-validator design means these tests need neither a database nor a live websocket, so they run fast and under the race detector alongside the rest of the suite.
