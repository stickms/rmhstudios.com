Each service ships as its own minimal container, built from a single parameterized multi-stage Dockerfile. Static Go binaries on a bare Alpine base keep images small (the gateway is ~33 MB) and the attack surface tiny.

One Dockerfile, every service

Rather than nine near-identical Dockerfiles, there is one, parameterized by a SERVICE build arg (which cmd/ to compile) and an optional EXTRA_PKGS arg for the few services that need OS packages at runtime. The build stage downloads modules with a cache mount, then compiles a fully static binary (CGO_ENABLED=0, -trimpath -ldflags="-s -w"). The runtime stage is Alpine with only what's required.

FROM golang:1.23-alpine AS build
ARG SERVICE
RUN --mount=type=cache,target=/go/pkg/mod go mod download
RUN CGO_ENABLED=0 GOOS=linux go build -trimpath -ldflags="-s -w" \
    -o /out/app ./cmd/${SERVICE}

FROM alpine:3.20 AS runtime
ARG EXTRA_PKGS=""
RUN apk add --no-cache ca-certificates tzdata ${EXTRA_PKGS} \
    && adduser -S app -G app
USER app
ENTRYPOINT ["/usr/local/bin/app"]

Two correctness details baked into the base image: ca-certificates (outbound TLS to Discord/DeepSeek) and tzdata (the doctrine-worker's timezone math). Both were easy to forget and would have produced subtle runtime failures; they're in the base for every service.

Security posture

  • Non-root: images run as an unprivileged app user.
  • Static binary: no libc surface, no shell needed at runtime (except where EXTRA_PKGS adds one for git).
  • Minimal layers: a single apk add, then the binary — small and auditable.
  • Reproducible: -trimpath strips local paths; module download is cache-mounted and verified against go.sum.

Verified build

The Dockerfile is not theoretical: the gateway image was built and run during verification. It produced a 33 MB image whose binary started, read its configuration, and exited cleanly with a structured error when DATABASE_URL was absent — confirming both stages work end to end.

$ docker build -f Dockerfile --build-arg SERVICE=gateway -t rmhstudios-go-gateway .
$ docker images rmhstudios-go-gateway   # 33.1MB
$ docker run --rm rmhstudios-go-gateway
{"level":"ERROR","msg":"config","service":"gateway",
 "error":"config: DATABASE_URL is required for service \"gateway\""}
