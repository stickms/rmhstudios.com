Every Go service in the fleet is one cmd/ binary built from a single module, github.com/rmhstudios/rmh-go, and stands on a small set of shared packages under pkg/. These libraries reproduce the conventions the legacy Node services already relied on — the same DATABASE_URL, the same log field shape, the same one-query session check — so the Go fleet drops in beside the React SSR tier rather than forking the platform. This section walks the foundation package by package, with real excerpts. The realtime hub that builds on pkg/events gets its own section; here we only describe the backplane primitive it consumes.

Scope of this layer. Everything under pkg/ described here is production-quality shared library code, not scaffolding: it compiles, passes go vet, and is covered by unit tests run under the race detector (concurrency matters — ratelimit, events and db are all touched by many goroutines at once). What is deliberately not reimplemented in Go is called out honestly per package: Better Auth session issuance, OAuth callbacks and cookie signing stay in the React tier; Go only validates the shared session table. Likewise only the subset of the Prisma schema the Go services own is modeled.

config — environment-driven configuration

The Helm chart from PR #121 already mounts .env.production as a Kubernetes Secret. Rather than invent new variable names, config reads the exact ones the Node fleet uses, so the same Secret feeds both tiers. LoadCommon assembles the shared struct and is the only required entry point; the GetX helpers handle service-specific extras.

// LoadCommon reads the shared configuration for a service.
func LoadCommon(serviceName string) (Common, error) {
    c := Common{
        ServiceName: serviceName,
        Env:         GetString("NODE_ENV", "production"),
        DatabaseURL: os.Getenv("DATABASE_URL"),
        DBPoolSize:  int32(GetInt("SERVER_DB_POOL_SIZE", 5)),
        RedisURL:    os.Getenv("REDIS_URL"),
        MetricsAddr: GetString("METRICS_ADDR", ":9090"),
        LogLevel:    GetString("LOG_LEVEL", "info"),
    }
    if c.DatabaseURL == "" {
        return c, fmt.Errorf("config: DATABASE_URL is required for service %q", serviceName)
    }
    return c, nil
}

The helpers — GetString, GetInt, GetBool, GetDuration (parses "30s"-style values), and GetCSV (splits and trims a comma list) — all treat an empty value as unset and fall back, so a blank env var never silently zeroes a setting. The Node PORT_* family is read the same way per-service with GetInt.

A missing DATABASE_URL is the one hard error in LoadCommon — services fail to boot rather than start half-configured, matching the Node clients' fail-fast posture.

log — slog JSON matching the Node logger

log is a thin wrapper over the standard library's log/slog. The goal is byte-compatible-enough JSON so existing log dashboards and alerts keep working. Two details make that happen: every line carries a service field, and the standard time key is renamed to timestamp to match server/shared/logger.ts.

func New(service, level string) *Logger {
    h := slog.NewJSONHandler(os.Stdout, &slog.HandlerOptions{
        Level: parseLevel(level),
        ReplaceAttr: func(_ []string, a slog.Attr) slog.Attr {
            // Match the Node logger's field name: "timestamp" not "time".
            if a.Key == slog.TimeKey {
                a.Key = "timestamp"
            }
            return a
        },
    })
    l := slog.New(h).With(slog.String("service", service))
    return &Logger{Logger: l, service: service}
}

The wrapper embeds *slog.Logger, so Info/Warn/Error/Debug pass straight through; With(...) returns a child that keeps the bound service name. Fatal logs at error level then os.Exit(1) — used only for unrecoverable startup failures, mirroring Node boot behavior. The logger is safe for concurrent use. A resulting line looks like:

{"timestamp":"2026-06-18T12:00:00Z","level":"INFO",
 "msg":"http listening","service":"gamehub","addr":":8081"}

db — pgxpool, fail-fast, and typed models

db standardizes the whole fleet on pgx/v5's pgxpool, replacing both shared/prisma-client.ts and the raw pg.Pool clients sprinkled through the Node services. Every query in every Go service runs through this one pool, so connection accounting is uniform.

Open: fail fast on an unreachable DB

func Open(ctx context.Context, dsn string, poolSize int32) (*DB, error) {
    cfg, err := pgxpool.ParseConfig(dsn)
    if err != nil { return nil, fmt.Errorf("db: parse DSN: %w", err) }
    if poolSize > 0 { cfg.MaxConns = poolSize }
    // Mirror the Node client's 30s idle / 10s connect timeouts.
    cfg.MaxConnIdleTime = 30 * time.Second
    cfg.ConnConfig.ConnectTimeout = 10 * time.Second

    pool, err := pgxpool.NewWithConfig(ctx, cfg)
    if err != nil { return nil, fmt.Errorf("db: create pool: %w", err) }
    pingCtx, cancel := context.WithTimeout(ctx, 10*time.Second)
    defer cancel()
    if err := pool.Ping(pingCtx); err != nil {
        pool.Close()
        return nil, fmt.Errorf("db: ping: %w", err)
    }
    return &DB{Pool: pool}, nil
}

The pool is sized from SERVER_DB_POOL_SIZE (via the caller), idle connections recycle after 30s, and connects time out at 10s — the same numbers the Node client used. The constructor pings before returning, so a service that cannot reach Postgres dies during startup instead of failing on its first request.

WaitForReachable: the 10×5s worker loop

Workers and the migrate job start before the DB is guaranteed up. The Node/deploy.sh "wait for DB reachability, 10×5s" loop becomes one reusable helper that polls Open with a single connection until it answers or attempts run out, and honors context cancellation between tries:

func WaitForReachable(ctx context.Context, dsn string, attempts int,
    interval time.Duration) (*DB, error) {
    var lastErr error
    for i := 0; i < attempts; i++ {
        d, err := Open(ctx, dsn, 1)
        if err == nil { return d, nil }
        lastErr = err
        select {
        case <-ctx.Done():
            return nil, ctx.Err()
        case <-time.After(interval):
        }
    }
    return nil, fmt.Errorf("db: not reachable after %d attempts: %w", attempts, lastErr)
}

auth — validating Better Auth sessions with one query

This is the single most important "we don't need the JS library" insight of the migration. The Node services already authenticate without the Better Auth SDK — they run one SQL lookup against the shared session table (see server/rmhbox/auth.ts and server/socket-server/index.ts). Go reproduces that lookup exactly. There is no Go port of Better Auth and none is needed: issuance, OAuth callbacks and cookie signing stay in the React tier; the Go fleet only trusts and validates the session table.

func (v *Validator) ValidateSession(ctx context.Context, token string) (Identity, error) {
    if token == "" { return Identity{}, ErrUnauthenticated }
    const q = `SELECT s."userId", s."expiresAt", u."name", u."image", u."isAdmin"
        FROM "session" s JOIN "user" u ON u."id" = s."userId"
        WHERE s."token" = $1 LIMIT 1`

    var (userID string; expiresAt time.Time
         name *string; image *string; isAdmin bool)
    err := v.pool.QueryRow(ctx, q, token).Scan(
        &userID, &expiresAt, &name, &image, &isAdmin)
    if err != nil {
        if errors.Is(err, pgx.ErrNoRows) { return Identity{}, ErrUnauthenticated }
        return Identity{}, err
    }
    if time.Now().After(expiresAt) { return Identity{}, ErrUnauthenticated }
    // ... populate Identity{UserID, Name, Image, IsAdmin}
}

One indexed lookup on session.token, joined to user, with an explicit application-side expiry check (time.Now().After(expiresAt)) so an expired-but-not-yet-purged row is rejected. An unknown token (pgx.ErrNoRows) and an expired token both collapse to the same ErrUnauthenticated sentinel — callers never learn why, which avoids leaking whether a token exists.

ResolveDiscordAccount covers the Discord Activity path: a Discord user id (already verified upstream against Discord's /users/@me) is mapped to a platform user via the account table where providerId = 'discord':

SELECT u."id", u."name", u."image", u."isAdmin"
FROM "account" a JOIN "user" u ON u."id" = a."userId"
WHERE a."providerId" = 'discord' AND a."accountId" = $1 LIMIT 1
No shared secret, no JWT verification, no duplicated cookie-signing logic. Because both tiers read the same table, a logout in the React app invalidates Go sessions instantly.

httpx — health, token extraction, graceful shutdown

httpx is the small set of HTTP primitives every service shares. It is the only pkg that imports another (log).

Health

Health(service, ready) returns the handler the Compose and Helm probes from PR #121 hit. With no readiness func it returns 200 {"status":"ok"}; with one, it runs the check under a 2s timeout and returns 503 {"status":"unavailable", ...} on failure — e.g. wiring a DB ping into readiness while keeping liveness cheap.

SessionToken — Bearer or better-auth cookie, token-before-dot

func SessionToken(r *http.Request) string {
    if h := r.Header.Get("Authorization"); strings.HasPrefix(h, "Bearer ") {
        return strings.TrimSpace(strings.TrimPrefix(h, "Bearer "))
    }
    for _, name := range []string{
        "better-auth.session_token", "__Secure-better-auth.session_token"} {
        if c, err := r.Cookie(name); err == nil && c.Value != "" {
            if i := strings.IndexByte(c.Value, '.'); i > 0 {
                return c.Value[:i]   // part before the signature dot
            }
            return c.Value
        }
    }
    return ""
}

Bearer header wins; otherwise it reads the Better Auth cookie (plain and __Secure- prefixed). The cookie value is <token>.<signature>, and Better Auth keys storage on the part before the dot — exactly what the session.token column holds — so SessionToken returns that prefix to feed straight into auth.ValidateSession.

Graceful shutdown

Server.Run(grace) starts the listener in a goroutine, then blocks on SIGINT/SIGTERM; on signal it calls Shutdown with a deadline to drain in-flight connections — the Go analog of the Node SIGTERM hooks (e.g. the discord-bot's 90s grace). A listener error short-circuits the wait. Workers without an HTTP listener use WaitForSignal(), which blocks on the same signals and returns the one received.

stop := make(chan os.Signal, 1)
signal.Notify(stop, syscall.SIGINT, syscall.SIGTERM)
select {
case err := <-errc:               // ListenAndServe failed
    return err
case sig := <-stop:               // SIGINT / SIGTERM
    s.logger.Info("shutdown signal received", "signal", sig.String())
}
ctx, cancel := context.WithTimeout(context.Background(), grace)
defer cancel()
return s.Shutdown(ctx)

ratelimit — concurrency-safe sliding window

This ports server/shared/rate-limit.ts: an in-memory sliding-window limiter keyed by (connection, event), bounded in size, with periodic GC of stale entries. The Node version was single-threaded; Go handlers run on many goroutines, so the port adds a sync.Mutex. A Rule is simply Max events per Window.

func (l *Limiter) Allow(conn, event string) bool {
    rule, ok := l.rules[event]
    if !ok { return true }               // unknown events allowed (Node default)
    now := time.Now().UnixNano()
    cutoff := now - rule.Window.Nanoseconds()
    key := conn + "|" + event

    l.mu.Lock(); defer l.mu.Unlock()
    b := l.buckets[key]
    if b == nil {
        if len(l.buckets) >= l.maxItems { return false }  // shed load, don't grow
        b = &bucket{}; l.buckets[key] = b
    }
    kept := b.hits[:0]                    // drop timestamps outside the window
    for _, t := range b.hits { if t >= cutoff { kept = append(kept, t) } }
    b.hits = kept; b.lastSeen = now
    if len(b.hits) >= rule.Max { return false }
    b.hits = append(b.hits, now)
    return true
}

Three safety properties worth calling out:

  • Bounded. When the bucket map hits maxItems (Node capped at 50k) a new key is refused rather than letting memory grow without limit.
  • Self-cleaning. A background sweep() ticks every 30s and evicts buckets idle longer than ttl, so disconnected connections don't linger.
  • Window pruning is in-place. kept := b.hits[:0] reuses the backing array, so steady-state Allow does no per-call allocation.

Forget(conn) drops every bucket for a connection on disconnect; Close() stops the sweeper and is idempotent via sync.Once.

telemetry — Prometheus per service

PR #121's roadmap lists observability as Stage 2. telemetry gives every Go service a uniform /metrics endpoint and a handful of standard instruments out of the box, so that stage is about wiring dashboards, not adding instrumentation. Each service gets its own prometheus.Registry (not the global default), with the service name attached as a const label on every metric.

func New(service string) *Metrics {
    reg := prometheus.NewRegistry()
    factory := promauto.With(reg)
    labels := prometheus.Labels{"service": service}
    return &Metrics{
        reg: reg,
        ActiveConnections: factory.NewGauge(prometheus.GaugeOpts{
            Name: "rmh_active_connections", ConstLabels: labels, ...}),
        MessagesTotal: factory.NewCounterVec(prometheus.CounterOpts{
            Name: "rmh_messages_total", ConstLabels: labels, ...}, []string{"event"}),
        DBQueries:  factory.NewCounterVec(..., []string{"outcome"}),
        JobRuns:    factory.NewCounterVec(..., []string{"job", "outcome"}),
    }
}

events — the cross-instance backplane

events is the pub/sub primitive that makes the realtime services horizontally scalable — the "socket.io Redis adapter + sticky sessions" work PR #121 flagged as Stage 1 and "the real first scaling investment". It is the piece that lets a realtime service run more than one replica. The realtime hub that wraps this bus with websocket fan-out is documented in its own section; here we cover the primitive only.

A single Bus interface has two implementations:

type Bus interface {
    Publish(ctx context.Context, topic string, payload []byte) error
    Subscribe(ctx context.Context, topic string) (<-chan Message, func(), error)
    Close() error
}
  • Local — in-process fan-out over a map[topic]→set of channels guarded by an RWMutex. Zero external deps; reproduces today's single-process behavior. Slow consumers are dropped (default: in the send select) because realtime favors freshness over buffering.
  • Redis — Redis pub/sub keyed by channel-per-topic, so N replicas share room broadcasts. Each Message carries an Origin (the publishing replica) so a subscriber can skip re-delivering to local connections that already got the message.

One function decides which the fleet runs — the single point that flips between single-replica and horizontally scaled realtime:

// FromURL returns a Redis bus when url is non-empty, otherwise a Local bus.
func FromURL(origin, url string) (Bus, error) {
    if url == "" {
        return NewLocal(origin), nil
    }
    return NewRedis(origin, url)
}

Because the decision keys on whether REDIS_URL is set, the same binary runs single-instance in dev (no Redis) and multi-replica in production (Redis present) with no code change — just the env var the Helm chart injects.

The Subscribe contract is identical for both buses — a receive channel plus an unsubscribe func that releases resources (closes the local channel, or Close()s the Redis subscription). The hub never branches on bus type; FromURL is the only place the choice is made.

Foundation summary

These eight packages are the entire contract between a Go service binary and the platform: read config the Node way, log the Node way, talk to one shared pool, trust the shared session table, expose health and metrics, rate-limit safely, and fan out events locally or across replicas. Every later service section assumes this layer and builds business logic on top of it.
