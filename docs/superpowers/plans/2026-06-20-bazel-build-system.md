# Bazel Build System Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give the repo a single-command, hermetic, incrementally-cached polyglot build/deploy by putting the Go services + container images under Bazel and fronting everything with a thin `Makefile` that also bootstraps its own toolchain.

**Architecture:** Bazel (bzlmod, via bazelisk pinned in `.bazelversion`) owns the hermetic core — Go binaries via `rules_go`+`gazelle`, container images via `rules_oci`. The Vite/TanStack Start frontend stays on pnpm, wrapped as one coarse non-hermetic Bazel leaf. A root `Makefile` is the ergonomic face (`make bootstrap/build/images/push/prod`) and orchestrates the edges (frontend leaf, Helm deploy). Backend source of truth is Go; the JS `server/*` services leave the default build path.

**Tech Stack:** Bazel 7.x (bzlmod) + bazelisk, rules_go 0.50.x, gazelle 0.39.x, rules_oci 2.x, rules_pkg 1.x, aspect_bazel_lib 2.x; Go 1.23 (hermetic SDK, no local install); pnpm + Vite (frontend leaf); Helm (deploy); GNU Make + bash (orchestration/bootstrap).

## Global Constraints

- Backend source of truth is **Go** (`go-services/`, module `github.com/rmhstudios/rmh-go`, go 1.23). JS `server/*` services are NOT a build target.
- Services (9): `gateway gamehub rmhmusic rmhtube rmhbox recap doctrine-worker vibe-worker discord-bot`.
- Image naming MUST match existing convention: `rmhstudios-go-<svc>:<SHA>`, optional `<REGISTRY>/` prefix; `SHA` = `git rev-parse --short HEAD` (fallback `dev`).
- Helm release `rmhstudios-go`, chart `deploy/helm/rmhstudios-go`, prod values `values-prod.yaml`, deployed with `--set image.tag=<SHA> --set image.registry=<REGISTRY> --atomic --wait`.
- Special runtime deps: `vibe-worker` → chromium (+ nss freetype harfbuzz ttf-freefont); `discord-bot` → git + openssh-client.
- `go` is NOT a required local install. `node`/`pnpm` required for the frontend leaf only.
- Bazel is invoked through **bazelisk**; never call a system `bazel` directly. Version pinned in `.bazelversion`.
- Frontend leaf is deliberately non-hermetic (reuses repo `node_modules`); tag its action `local`.
- Commit after every task. Never commit a broken `bazel build //go-services/...`.

## Task 5: Author the `go_service_image` macro

**Files:**
- Create: `bazel/defs.bzl`, `bazel/BUILD.bazel`

**Interfaces:**
- Produces: `go_service_image(name, binary, base, registry_repo)` macro generating `<name>_image`, `<name>_load`, `<name>_push` targets. Consumed by Task 6.

- [ ] **Step 1: Create the macro**

Create `bazel/defs.bzl`:
```python
"""Reusable container-image rules for the Go service fleet."""

load("@rules_oci//oci:defs.bzl", "oci_image", "oci_load", "oci_push")
load("@rules_pkg//pkg:tar.bzl", "pkg_tar")
load("@aspect_bazel_lib//lib:transitions.bzl", "platform_transition_filegroup")

def go_service_image(name, binary, base = "@distroless_base", registry_repo = None, platform = "@rules_go//go/toolchain:linux_amd64"):
    """Build a per-service OCI image from a go_binary.

    Args:
      name: service name (e.g. "gateway").
      binary: label of the go_binary (e.g. "//go-services/cmd/gateway:gateway").
      base: base OCI image label.
      registry_repo: full push repository (e.g. "ghcr.io/owner/rmhstudios-go-gateway").
      platform: target platform for the binary inside the image.
    """

    # Force the binary to the Linux container platform regardless of host.
    platform_transition_filegroup(
        name = name + "_linux_bin",
        srcs = [binary],
        target_platform = platform,
    )

    pkg_tar(
        name = name + "_layer",
        srcs = [":" + name + "_linux_bin"],
        package_dir = "/usr/local/bin",
    )

    oci_image(
        name = name + "_image",
        base = base,
        tars = [":" + name + "_layer"],
        entrypoint = ["/usr/local/bin/" + binary.rsplit(":", 1)[-1]],
        env = {"PLAYWRIGHT_CHROMIUM_EXECUTABLE_PATH": "/usr/bin/chromium-browser"},
    )

    oci_load(
        name = name + "_load",
        image = ":" + name + "_image",
        repo_tags = ["rmhstudios-go-" + name + ":dev"],
    )

    if registry_repo:
        oci_push(
            name = name + "_push",
            image = ":" + name + "_image",
            repository = registry_repo,
        )
```

Create `bazel/BUILD.bazel`:
```python
exports_files(["defs.bzl"])
```

## Task 8: Bootstrap script (dependency self-install + guard)

**Files:**
- Create: `scripts/preflight.sh`

**Interfaces:**
- Produces: `scripts/preflight.sh [--guard]` — `make bootstrap` runs full install; `--guard` is the fast check run before build targets.

- [ ] **Step 1: Write preflight.sh**

Create `scripts/preflight.sh`:
```bash
#!/usr/bin/env bash
# Toolchain bootstrap + guard for the Bazel build system.
#   scripts/preflight.sh           # full bootstrap (install missing)
#   scripts/preflight.sh --guard   # fast check; install cheap, hard-fail on heavy
set -euo pipefail

GUARD=0; [ "${1:-}" = "--guard" ] && GUARD=1
OS="$(uname -s)"
have() { command -v "$1" >/dev/null 2>&1; }
say() { printf '\033[1;36m[preflight]\033[0m %s\n' "$*"; }
die() { printf '\033[1;31m[preflight]\033[0m %s\n' "$*" >&2; exit 1; }

pkg_install() {
  case "$OS" in
    Darwin) have brew || die "Homebrew required: https://brew.sh"; brew install "$1" ;;
    Linux)
      if have apt-get; then sudo apt-get update -y && sudo apt-get install -y "$1";
      elif have apk; then sudo apk add --no-cache "$1";
      else die "No supported package manager (apt/apk) found"; fi ;;
    *) die "Unsupported OS: $OS" ;;
  esac
}

# bazelisk — the one true entrypoint.
if ! have bazelisk && ! have bazel; then
  say "installing bazelisk"
  case "$OS" in Darwin) pkg_install bazelisk ;; Linux) pkg_install bazelisk || pkg_install bazel ;; esac
fi

# node — required for the frontend leaf. Heavy runtime: install on bootstrap, guide on guard.
if ! have node; then
  if [ "$GUARD" = 1 ]; then die "node missing — install Node 20+ then re-run (e.g. 'brew install node')"; fi
  pkg_install node
fi

# pnpm via corepack (ships with node).
if ! have pnpm; then say "enabling pnpm via corepack"; corepack enable pnpm || die "corepack enable failed"; fi

# helm — only needed at deploy.
if ! have helm; then
  if [ "$GUARD" = 1 ]; then say "helm missing (only needed for 'make prod')"; else pkg_install helm; fi
fi

# docker — daemon cannot be auto-provisioned; hard-fail with guidance.
if ! have docker; then die "docker missing — install Docker Desktop / Engine, then re-run"; fi
if ! docker info >/dev/null 2>&1; then die "docker daemon not running — start Docker, then re-run"; fi

say "toolchain OK"
```

- [ ] **Step 2: Make it executable + run the guard**

Run: `chmod +x scripts/preflight.sh && ./scripts/preflight.sh --guard`
Expected: `[preflight] toolchain OK` (bazelisk/node/pnpm/docker all present in this env; helm present).

- [ ] **Step 3: Commit**

```bash

git commit -m "build: add preflight bootstrap + guard script"
```

```make
# rmhstudios.com — single-command build/deploy entrypoint.
# Bazel (via bazelisk) owns the hermetic core; this Makefile is the ergonomic face.

BAZEL        ?= bazelisk
SERVICES     := gateway gamehub rmhmusic rmhtube rmhbox recap doctrine-worker vibe-worker discord-bot
SHA          := $(shell git rev-parse --short HEAD 2>/dev/null || echo dev)
REGISTRY     ?=
RELEASE      ?= rmhstudios-go
CHART        := deploy/helm/rmhstudios-go
GUARD        := ./scripts/preflight.sh --guard

.PHONY: bootstrap dev gazelle build images push prod test test-e2e clean help

help: ## list targets
	@grep -hE '^[a-z-]+:.*?## ' $(MAKEFILE_LIST) | awk -F':.*?## ' '{printf "  \033[36m%-12s\033[0m %s\n",$$1,$$2}'

bootstrap: ## install/verify all toolchain prerequisites
	./scripts/preflight.sh

dev: ## run the existing pnpm dev loop (Bazel not involved)
	pnpm run dev

gazelle: ## regenerate Go BUILD.bazel files
	$(BAZEL) run //:gazelle

build: ## build artifacts: all Go binaries + frontend bundle
	@$(GUARD)
	$(BAZEL) build //go-services/... //:frontend

images: ## build + load every service image into Docker
	@$(GUARD)
	@for svc in $(SERVICES); do \
		echo "==> loading rmhstudios-go-$$svc"; \
		$(BAZEL) run //go-services/cmd/$$svc:$${svc}_load || exit 1; \
	done
	$(BAZEL) run //:frontend_load

push: ## push every service image to $REGISTRY (REGISTRY must be set)
	@test -n "$(REGISTRY)" || { echo "REGISTRY is required for push"; exit 1; }
	@$(GUARD)
	@for svc in $(SERVICES); do \
		echo "==> pushing $(REGISTRY)/rmhstudios-go-$$svc:$(SHA)"; \
		$(BAZEL) run //go-services/cmd/$$svc:$${svc}_push -- \
			--repository=$(REGISTRY)/rmhstudios-go-$$svc --tag=$(SHA) || exit 1; \
	done

prod: images ## build images, push, and deploy to the cluster via Helm
	@$(GUARD)
	@if [ -n "$(REGISTRY)" ]; then $(MAKE) push REGISTRY=$(REGISTRY); fi
	helm upgrade --install $(RELEASE) $(CHART) \
		-f $(CHART)/values-prod.yaml \
		--set image.tag=$(SHA) --set image.registry=$(REGISTRY) \
		--atomic --wait --timeout 5m

test: ## run Go tests (Bazel) + frontend tests (vitest)
	@$(GUARD)
	$(BAZEL) test //go-services/...
	pnpm test

test-e2e: ## run the Go end-to-end suite
	cd go-services && bash scripts/e2e/run.sh

clean: ## remove Bazel outputs
	$(BAZEL) clean
```
