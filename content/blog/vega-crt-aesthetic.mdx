## Finding Vega's Visual Identity

Every game needs a look that's instantly recognizable. For Project Vega, I wanted something that felt *wrong* in a specific, deliberate way — like a hospital monitoring system running corrupted firmware. I call it the **Clinical Glitch** aesthetic: CRT scanlines, chromatic aberration, terminal fonts, medical grid patterns, and a palette restricted to sickly greens, warning ambers, and sterile whites against deep black.

The inspiration came from staring at an old oscilloscope in a thrift store. That flickering green trace on the dark screen — it looked alive. It looked like it was *trying to tell me something*. That feeling is what I wanted Vega to evoke every second of gameplay.

## The Post-Processing Pipeline

All of Vega's visual effects are applied as **post-processing passes** on the HTML5 Canvas. The game renders the clean scene first (towers, enemies, projectiles, grid), then the post-processing pipeline layers effects on top:

```typescript
class PostProcessor {
  private effects: PostEffect[] = [];

  constructor() {
    this.effects = [
      new ScanlineEffect(),
      new ChromaticAberrationEffect(),
      new VignetteEffect(),
      new GlitchEffect(),
      new CRTCurveEffect(),
    ];
  }

  apply(ctx: CanvasRenderingContext2D, sanity: number) {
    const imageData = ctx.getImageData(0, 0, ctx.canvas.width, ctx.canvas.height);
    let processed = imageData;

    for (const effect of this.effects) {
      processed = effect.process(processed, sanity);
    }

    ctx.putImageData(processed, 0, 0);
  }
}
```

The `sanity` parameter is crucial — as the player's Sanity resource drops, the visual effects intensify. At 100% Sanity, the scanlines are subtle and the screen is clean. At 30% Sanity, the glitches are aggressive, chromatic aberration is severe, and the entire image feels like it's barely holding together.

## Scanline Effect

CRT scanlines are the foundation of the aesthetic. Real CRT displays rendered every other horizontal line in alternating fields, creating the characteristic horizontal banding.

My scanline implementation is deliberately imperfect — real CRTs weren't uniform:

```typescript
class ScanlineEffect implements PostEffect {
  process(imageData: ImageData, sanity: number): ImageData {
    const { data, width, height } = imageData;
    const intensity = 0.15 + (1 - sanity) * 0.2;  // stronger at low sanity
    const lineSpacing = 2;

    for (let y = 0; y < height; y++) {
      if (y % lineSpacing === 0) continue;

      // Slight variation in darkness per line
      const lineDark = intensity * (0.85 + Math.random() * 0.15);

      for (let x = 0; x < width; x++) {
        const i = (y * width + x) * 4;
        data[i]     *= (1 - lineDark);  // R
        data[i + 1] *= (1 - lineDark);  // G
        data[i + 2] *= (1 - lineDark);  // B
      }
    }

    return imageData;
  }
}
```

The random variation per line is key. Perfectly uniform scanlines look digital and flat. Adding just 15% brightness variation makes them feel analog and alive — like a real cathode ray tube with slight inconsistencies in the electron gun.

### The Phosphor Glow

I also simulate the **phosphor persistence** of CRT displays. On a real CRT, bright elements leave a brief afterglow as the phosphor coating decays. I achieve this by blending the current frame with a dimmed copy of the previous frame:

```typescript
class PhosphorGlow {
  private previousFrame: ImageData | null = null;
  private decayRate = 0.85;

  blend(current: ImageData): ImageData {
    if (!this.previousFrame) {
      this.previousFrame = new ImageData(new Uint8ClampedArray(current.data), current.width);
      return current;
    }

    for (let i = 0; i < current.data.length; i += 4) {
      const ghostR = this.previousFrame.data[i] * this.decayRate;
      const ghostG = this.previousFrame.data[i + 1] * this.decayRate;
      const ghostB = this.previousFrame.data[i + 2] * this.decayRate;

      current.data[i]     = Math.max(current.data[i], ghostR);
      current.data[i + 1] = Math.max(current.data[i + 1], ghostG);
      current.data[i + 2] = Math.max(current.data[i + 2], ghostB);
    }

    this.previousFrame = new ImageData(new Uint8ClampedArray(current.data), current.width);
    return current;
  }
}
```

This creates gorgeous trailing effects on moving objects. Projectiles leave green streaks. Enemies smear slightly as they move. It's subtle but transformative — it makes the whole game feel like it's being displayed on aging hardware.

## Chromatic Aberration

Chromatic aberration — the separation of color channels simulating lens distortion — is Vega's most recognizable visual effect. Each color channel (red, green, blue) is offset by a few pixels, creating a prismatic fringing effect:

```typescript
class ChromaticAberrationEffect implements PostEffect {
  process(imageData: ImageData, sanity: number): ImageData {
    const { data, width, height } = imageData;
    const offset = Math.floor(2 + (1 - sanity) * 4);  // 2-6px based on sanity
    const output = new ImageData(width, height);

    for (let y = 0; y < height; y++) {
      for (let x = 0; x < width; x++) {
        const i = (y * width + x) * 4;

        // Red channel shifted left
        const rX = Math.max(0, x - offset);
        const rI = (y * width + rX) * 4;
        output.data[i] = data[rI];

        // Green channel stays centered
        output.data[i + 1] = data[i + 1];

        // Blue channel shifted right
        const bX = Math.min(width - 1, x + offset);
        const bI = (y * width + bX) * 4;
        output.data[i + 2] = data[bI + 2];

        output.data[i + 3] = data[i + 3];
      }
    }

    return output;
  }
}
```

At full Sanity, the offset is a barely-perceptible 2 pixels — just enough to give edges that signature CRT fuzziness. As Sanity plummets, the offset grows to 6 pixels, and the game world literally starts coming apart at the seams. Colors bleed. Text becomes unreadable. It's deeply uncomfortable, and that's exactly the point.

## The Glitch System

Glitch effects are triggered stochastically, with frequency scaling inversely with Sanity. They come in three varieties:

### Horizontal Tear

A random horizontal band of the screen is shifted left or right by a random pixel amount. This simulates a signal synchronization failure:

```typescript
const horizontalTear = (ctx: CanvasRenderingContext2D) => {
  const y = Math.random() * ctx.canvas.height;
  const sliceHeight = 5 + Math.random() * 20;
  const shift = (Math.random() - 0.5) * 40;

  const slice = ctx.getImageData(0, y, ctx.canvas.width, sliceHeight);
  ctx.putImageData(slice, shift, y);
};
```

### Color Inversion Flash

A brief, single-frame inversion of a rectangular region. It's jarring and disorienting — perfect for those moments when the player is losing control.

### Static Noise Blocks

Small rectangles filled with random pixel data, simulating signal noise. They appear and disappear within a frame or two, creating the impression of data corruption.

The glitch system has a **cooldown** to prevent effects from becoming constant at low Sanity. Even at 10% Sanity, glitches fire no more than once every 2 seconds. Enough to be stressful, not enough to be unplayable.

## The Color Palette

Vega's palette is intentionally restricted:

- **Primary green (#00ff41):** Terminal text, Synapse projectiles, healthy UI elements
- **Warning amber (#ffb000):** Alerts, Suppressor effects, caution states
- **Sterile white (#e0e0e0):** Grid lines, structural elements
- **Deep black (#0a0a0a):** Background, negative space
- **Corruption red (#ff0040):** Damage indicators, critical warnings

This limited palette does two things. First, it makes the game instantly readable — green is good, amber is caution, red is danger. Second, it reinforces the medical/terminal aesthetic. This is a diagnostic tool, not a theme park ride. The restraint in color makes the rare splash of corruption red genuinely alarming.

## Performance Considerations

Pixel-level manipulation on Canvas is expensive. Processing every pixel of a 1280x720 canvas means touching nearly a million pixels per effect, per frame. At 60fps, that's 60 million pixel operations per second.

I optimize with several strategies. Effects are applied at half resolution and upscaled. Only the scanline and vignette effects run every frame — chromatic aberration and glitches run on alternating frames. The phosphor glow uses a single-buffer approach instead of creating new ImageData objects. These optimizations keep the post-processing pipeline under 4ms per frame on mid-range hardware, leaving plenty of headroom for game logic and rendering.

## The Feeling It Creates

When everything comes together — scanlines humming, phosphor trailing behind projectiles, the edges of the screen softly vignetted, a glitch tearing across the display as an Anxiety Sprite breaches your defenses — Vega looks like nothing else I've played. It feels like you're peering into a machine that's barely maintaining coherence. And as your Sanity drops, the machine starts to fail.

That's the Clinical Glitch aesthetic. Not just a look — a feeling. Unease wrapped in nostalgia. Medical precision dissolving into digital entropy. It's exactly where I want Vega to live.
